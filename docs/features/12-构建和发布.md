## 12.1 构建配置

### tsup 配置

```typescript
// tsup.config.ts
import { defineConfig } from 'tsup';

export default defineConfig({
  entry: ['src/index.ts'],
  format: ['cjs', 'esm'],
  dts: true,
  splitting: false,
  sourcemap: true,
  clean: true,
  treeshake: true,
  minify: false,
  target: 'node18',
  outDir: 'dist',
  outExtension({ format }) {
    if (format === 'cjs') return { js: '.js' };
    if (format === 'esm') return { js: '.mjs' };
    return { js: '.js' };
  },
  esbuildOptions(options) {
    options.jsx = 'automatic';
    options.jsxImportSource = 'react';
    options.define = {
      'process.env.NODE_ENV': JSON.stringify(process.env.NODE_ENV || 'development'),
    };
  },
  external: [
    'react',
    'react-dom',
    'ink',
    'chalk',
    'commander',
    'fs-extra',
    'date-fns',
    'cli-table3',
    'ora',
    'jsonstream',
  ],
});
```

### 构建输出

```
dist/
├── index.js          # CommonJS 格式
├── index.mjs         # ESM 格式
├── index.d.ts        # TypeScript 类型定义
└── index.d.mts       # ESM 类型定义
```

## 12.4 版本管理

### 语义化版本

```
MAJOR.MINOR.PATCH

MAJOR - 不兼容的 API 更改
MINOR - 向后兼容的新功能
PATCH - 向后兼容的问题修复
```

### 版本更新

```bash
# 更新补丁版本 (1.0.0 -> 1.0.1)
npm version patch

# 更新次版本 (1.0.0 -> 1.1.0)
npm version minor

# 更新主版本 (1.0.0 -> 2.0.0)
npm version major

# 预发布版本 (1.0.0 -> 1.0.1-0)
npm version prerelease
```

### 版本脚本

```json
{
  "scripts": {
    "version": "npm run build && git add -A",
    "postversion": "git push && git push --tags"
  }
}
```

## 12.7 GitHub Release

### 创建 Release

1. **推送标签**
```bash
git push origin v1.0.0
```

2. **在 GitHub 创建 Release**
   - 访问 GitHub 仓库
   - 点击 "Releases"
   - 点击 "Create a new release"
   - 选择标签
   - 填写标题和描述
   - 上传构建产物

### Release 脚本

```bash
#!/bin/bash
# release.sh

# 1. 运行测试
npm test

# 2. 构建
npm run build

# 3. 更新版本
npm version $1

# 4. 推送
git push && git push --tags

# 5. 发布到 npm
npm publish

echo "Release $1 completed!"
```

## 12.8 CI/CD 配置

### GitHub Actions

```yaml
# .github/workflows/release.yml
name: Release

on:
  push:
    tags:
      - 'v*'

jobs:
  release:
    runs-on: ubuntu-latest
    
    steps:
      - uses: actions/checkout@v3
      
      - name: Setup Node.js
        uses: actions/setup-node@v3
        with:
          node-version: '18'
          registry-url: 'https://registry.npmjs.org'
      
      - name: Install dependencies
        run: npm ci
      
      - name: Run tests
        run: npm test
      
      - name: Build
        run: npm run build
      
      - name: Publish to npm
        run: npm publish
        env:
          NODE_AUTH_TOKEN: ${{ secrets.NPM_TOKEN }}
      
      - name: Create GitHub Release
        uses: actions/create-release@v1
        env:
          GITHUB_TOKEN: ${{ secrets.GITHUB_TOKEN }}
        with:
          tag_name: ${{ github.ref }}
          release_name: Release ${{ github.ref }}
          draft: false
          prerelease: false
```

## 12.11 故障排除

### 常见问题

1. **安装失败**
   ```bash
   # 清理缓存
   npm cache clean --force
   
   # 重新安装
   npm install -g codex-session-viewer
   ```

2. **命令未找到**
   ```bash
   # 检查 npm 全局路径
   npm config get prefix
   
   # 添加到 PATH
   export PATH="$(npm config get prefix)/bin:$PATH"
   ```

3. **权限错误**
   ```bash
   # 使用 sudo (Linux/macOS)
   sudo npm install -g codex-session-viewer
   
   # 或配置 npm 使用其他目录
   npm config set prefix ~/.npm-global
   ```

4. **版本冲突**
   ```bash
   # 卸载旧版本
   npm uninstall -g codex-session-viewer
   
   # 安装最新版本
   npm install -g codex-session-viewer@latest
   ```
